# 실습과제 1

# IDE란 무엇인가?

IDE(Integrated Development Environment, 통합 개발 환경)는 프로그래머가 소프트웨어를 개발하는 데 필요한 모든 도구를 하나의 인터페이스로 통합하여 제공하는 소프트웨어 애플리케이션입니다.

과거에는 코드를 작성하는 '편집기(Editor)', 기계어로 번역하는 '컴파일러(Compiler)', 실행 파일을 만드는 '링커(Linker)', 오류를 수정하는 '디버거(Debugger)'를 각각 따로 실행해야 했습니다. 

하지만 IDE는 이 모든 과정을 하나의 프로그램 안에서 처리할 수 있게 해주어 개발 효율성과 생산성을 극대화합니다.


# IDE의 종류

### Visual Studio: 
Microsoft에서 개발한 IDE로, Windows 환경에서 가장 강력하고 널리 사용됩니다. '명품 C++ Programming' 교재의 실습도 주로 이 환경을 기준으로 설명되어 있습니다.

### Eclipse (CDT): 

자바 개발용으로 유명하지만, C/C++ Development Tooling(CDT) 플러그인을 설치하면 훌륭한 C++ IDE로 활용할 수 있습니다. 다양한 운영체제를 지원합니다.

### CLion: 

JetBrains사에서 만든 상용 C++ 전용 IDE입니다. 코드 분석 및 리팩토링 기능이 매우 뛰어나 실무에서 선호도가 높습니다.

### Xcode: 

Apple에서 제공하는 IDE로, macOS 환경에서 C++ 및 다양한 언어를 개발할 때 필수적으로 사용됩니다.

### Code::Blocks / Dev-C++: 

비교적 가볍고 무료로 사용할 수 있어 교육용으로 꾸준히 활용되는 IDE입니다.


# IDE에 포함된 핵심 기능
### 소스 코드 편집기 (Source Code Editor): 

C++ 문법에 맞게 색상을 다르게 표시해 주는 구문 강조(Syntax Highlighting) 기능, 코드 자동 완성 기능 등을 제공하여 오타를 줄이고 빠르게 코드를 작성할 수 있게 돕습니다.

### 빌드 도구 (Build Automation Tools): 

클릭 한 번으로 컴파일러와 링커를 연속적으로 실행하여, 인간이 읽을 수 있는 소스 코드(.cpp)를 컴퓨터가 이해할 수 있는 기계어 코드(.obj 등)로 번역하고, 필요한 라이브러리와 연결하여 최종 실행 파일(.exe)을 생성합니다.

### 디버거 (Debugger): 

프로그램 실행 중 발생하는 논리적 오류를 찾아낼 수 있는 도구입니다. 코드를 한 줄씩 실행(Step-over/Step-into)해 보거나, 특정 지점에서 실행을 멈추고(Breakpoint) 변수에 저장된 메모리 값을 실시간으로 확인할 수 있습니다.



### 프로젝트 관리 (Project Management): 

여러 개의 헤더 파일(.h)과 소스 파일(.cpp), 리소스들을 하나의 프로젝트 단위로 묶어 체계적으로 관리할 수 있도록 계층적인 파일 탐색 기능을 제공합니다.
