# 실습과제 1

### 1. `string` 클래스의 자동 크기 조절 원리 (동적 할당과 재할당)

C언어 스타일의 `char` 배열은 생성할 때 크기를 고정해야 하지만, C++의 `std::string` 클래스는 사용자가 입력하는 문자열의 길이에 맞춰 내부적으로 동적 메모리 할당(Dynamic Memory Allocation)을 수행합니다. 

그 동작 원리는 다음과 같습니다.

* **크기(Size)와 용량(Capacity):** `string` 객체는 내부적으로 실제 저장된 문자열의 길이(`size`)와, 앞으로의 입력을 대비해 메모리에 여유 있게 미리 할당해 둔 전체 공간의 크기(`capacity`)를 따로 관리합니다.
* **재할당(Reallocation):** 사용자가 문자를 계속 입력하여 현재 할당된 용량(`capacity`)이 가득 차게 되면, `string` 클래스는 내부적으로 기존 용량보다 더 큰(보통 1.5배에서 2배) 새로운 메모리 공간을 `new` 연산자로 할당받습니다.
* **데이터 복사 및 메모리 해제:** 기존 메모리에 있던 문자열을 새로 할당받은 넓은 공간으로 복사한 뒤, 기존의 좁은 메모리 공간은 `delete` 연산자를 통해 운영체제에 반환합니다.

이 모든 복잡한 메모리 관리 과정이 클래스 내부에 캡슐화(Encapsulation)되어 자동으로 이루어집니다. 따라서 개발자는 `char` 배열에서 흔히 발생하는 버퍼 오버플로우(Buffer Overflow, 배열의 크기를 넘어서 메모리를 침범하는 오류)를 걱정할 필요가 없습니다. 이것이 바로 C++ 프로그래밍에서 고정 크기 `char` 배열 대신 `string` 클래스를 적극적으로 사용해야 하는 가장 큰 이유입니다.

### 2. 엔터키(개행 문자)와 `getline` 함수의 구분 문자(Delimiter)의 차이

이 질문은 입력 버퍼(Input Buffer)가 어떻게 처리되는지 이해하는 데 매우 중요합니다.

* **엔터키 (`\n`):** 사용자가 키보드로 입력을 마치고 엔터키를 누르면, 입력 버퍼에는 개행 문자(`\n`)가 들어갑니다. 일반적인 `cin >> string` 연산은 공백(띄어쓰기, 탭, 개행 문자)을 기준으로 입력을 끊기 때문에, 띄어쓰기가 포함된 문장을 한 번에 입력받을 수 없습니다.
* **`getline` 함수의 구분 문자 (Delimiter):** `getline(cin, str, 구분문자)` 함수는 세 번째 인자로 전달된 '구분 문자'를 만날 때까지 띄어쓰기를 포함한 버퍼의 모든 내용을 읽어 들입니다.

이 둘의 차이점은 **입력 종료의 주도권**이 어디에 있느냐에 따라 명확해집니다.

1.  **기본 동작:** 만약 `getline(cin, str)`처럼 구분 문자를 생략하면, 디폴트 구분 문자는 바로 엔터키(`\n`)가 됩니다. 즉, 사용자가 엔터키를 치면 그 줄의 끝까지를 하나의 문자열로 읽고 입력을 종료합니다.
2.  **구분 문자를 명시했을 때:** 예를 들어 `getline(cin, str, ';')` 처럼 세미콜론을 구분 문자로 지정했다고 가정해 봅시다. 이때 사용자가 키보드로 문자를 치다가 **엔터키를 누르더라도 입력은 종료되지 않습니다.** 엔터키(`\n`)조차 단순한 문자열의 일부로 `str`에 저장되며, 오직 키보드로 `;` (세미콜론)을 입력해야만 비로소 입력이 완전히 종료됩니다.
