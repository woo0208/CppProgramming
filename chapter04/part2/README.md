# 실습과제 2
#### 1. 매크로 상수 NULL
* **정의의 차이 (C vs C++)**: 
  * **C 언어**: 주로 `(void*)0`으로 정의됩니다. C에서는 `void*`가 다른 포인터 타입으로 암시적 변환이 가능하기 때문입니다.
  * **C++ 언어**: C++에서는 `void*`의 암시적 형변환을 허용하지 않기 때문에, 단순히 정수 `0` 또는 `0L`로 정의됩니다.
* **정의된 헤더 파일**: `<iostream>`, `<cstddef>` 등 여러 표준 C++ 헤더에 포함되어 있습니다. (C 언어의 경우 `<stddef.h>`, `<stdio.h>` 등)
* **사용 용도**: 포인터 변수가 현재 아무런 유효한 메모리 주소도 가리키지 않고 있음을 명시적으로 나타낼 때 사용합니다. (예: `int* p = NULL;`)

#### 2. 널 문자('\0')와 NULL의 차이
* **정의**: 
  * **널 문자**: 문자열(C-스트링)의 끝을 나타내기 위해 사용하는 이스케이프 시퀀스 문자입니다.
  * **NULL**: 포인터가 비어있음을 나타내는 매크로 상수입니다.
* **자료형**: 널 문자는 문자형(`char`)이고, NULL은 C++에서 정수형(`int`)으로 처리됩니다.
* **메모리 크기**: 널 문자는 1바이트를 차지합니다. NULL 상수 자체는 컴파일러에 따라 int 크기(보통 4바이트)를 가지며, 포인터 변수에 대입될 때는 시스템(32비트/64비트)에 따라 4바이트 또는 8바이트의 포인터 크기를 가집니다.
* **실제 값**: 둘 다 컴퓨터 메모리 상에 저장되는 실제 정수 값은 **0**으로 동일합니다.
* **사용 용도**: 널 문자는 `char` 배열 끝에 문자열의 종료를 알리기 위해 사용하고, NULL은 포인터 변수를 초기화하거나 검사할 때 사용합니다.

#### 3. nullptr (C++11 이후)
* **정의**: C++11 표준에서 새롭게 도입된 **포인터 리터럴(Pointer Literal)** 키워드입니다. 
* **도입 이유 (NULL의 문제점 해결)**: 기존 C++에서 `NULL`은 정수 `0`으로 처리되었기 때문에, 함수 오버로딩 시 정수를 매개변수로 받는 함수와 포인터를 매개변수로 받는 함수가 겹칠 때 모호성이 발생했습니다. (예: `func(int)`와 `func(int*)`가 있을 때 `func(NULL)`을 호출하면 의도치 않게 `func(int)`가 호출됨)
* **특징**: `nullptr`은 그 자체로 `std::nullptr_t`라는 명확한 포인터 타입을 가지므로, 어떤 포인터 타입으로도 형변환이 가능하지만 정수형으로는 변환되지 않아 오버로딩 버그를 완벽하게 방지합니다. 따라서 현대 C++에서는 `NULL` 대신 반드시 `nullptr`을 사용해야 합니다.
