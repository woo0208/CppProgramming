# 실습과제 1


### 1. 배열명의 의미
C++에서 배열의 이름은 해당 배열이 할당된 연속적인 메모리 공간의 시작 주소(Base Address)를 가리키는 상수 포인터(Constant Pointer)로 해석됩니다. 
* 즉, `int arr[5];`로 배열을 선언했을 때 배열명 `arr`은 첫 번째 원소인 `arr[0]`의 메모리 주소(`&arr[0]`)와 동일한 값을 가집니다. 단, 상수 포인터이므로 배열명 자체에 다른 메모리 주소값을 대입하여 변경할 수는 없습니다.

### 2. 객체 포인터의 의미
객체 포인터란 특정 클래스 타입으로 생성된 **객체(Instance)가 메모리에 할당된 시작 주소를 저장하는 포인터 변수**입니다.
* 기본 자료형(int, double 등)의 포인터와 동일한 원리로 동작하지만, 대상이 객체라는 점이 다릅니다.
* 객체 포인터를 사용하면 화살표 멤버 접근 연산자(`->`)를 통해 해당 객체의 `public` 멤버 변수나 멤버 함수에 직접 접근하고 호출할 수 있습니다. (예: `Circle* p = &donut; p->getArea();`)

### 3. 포인터를 이용하여 배열 각 원소의 주소를 구하는 방법
포인터 연산(Pointer Arithmetic)을 활용하여 각 원소의 주소를 구할 수 있습니다.
* 포인터 변수 `p`가 배열의 시작 주소를 가리키고 있을 때, `p + i` (i는 0 이상의 정수 인덱스)를 계산하면 배열의 i번째 원소의 주소가 됩니다.
* 여기서 `+ i`는 단순히 정수 i를 더하는 것이 아니라, 포인터가 선언된 **자료형의 크기(바이트) × i** 만큼 메모리 주소를 이동한다는 의미입니다. 
* 예: `int`형 포인터(일반적으로 int는 4바이트) `p`에서 `p + 1`을 수행하면, 실제로는 p의 주소값보다 4바이트 증가한 다음 원소의 메모리 주소를 가리키게 됩니다.

### 4. 포인터를 이용하여 배열 각 원소의 값을 구하는 방법
포인터 연산으로 계산된 주소에 역참조 연산자(Dereference Operator, `*`)를 사용하여 해당 주소 공간에 저장된 실제 값을 읽어올 수 있습니다.
* 주소를 구하는 수식 `(p + i)` 전체를 괄호로 묶고 앞에 역참조 연산자를 붙여 `*(p + i)` 형태로 작성하면 i번째 원소의 값을 구할 수 있습니다.
* 이는 C++의 인덱스 연산자를 사용한 `p[i]` 와 내부적으로 완벽하게 동일한 동작을 수행합니다. 컴파일러는 `p[i]` 형태의 코드를 만나면 자동으로 `*(p + i)`로 변환하여 처리합니다.
