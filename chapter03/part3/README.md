# 실습과제 1

### 1. 객체의 생성과 소멸의 의미

객체 지향 프로그래밍(OOP)에서 객체의 생성과 소멸은 단순히 변수를 하나 선언하고 지우는 것을 넘어, **메모리 할당과 자원 관리의 자동화**를 의미합니다.

* **객체의 생성 (Creation & Initialization):**
    * **의미:** 객체가 저장될 물리적인 **메모리 공간을 할당**받고, 생성자(Constructor)가 호출되어 객체의 멤버 변수들을 유효한 상태로 초기화(Initialization)하는 과정입니다.
    * **특징:** 생성자가 성공적으로 실행을 마쳐야만 비로소 완전한 '객체(인스턴스)'로서 기능할 수 있습니다.
* **객체의 소멸 (Destruction & Deallocation):**
    * **의미:** 객체의 수명(Lifetime)이 다했을 때, 메모리에서 객체가 해제되기 직전 소멸자(Destructor)가 호출되어 객체가 점유하고 있던 자원을 반환(Resource Release)하는 과정입니다.
    * **특징:** 내부에서 동적으로 할당(new)한 메모리나 열어둔 파일, 네트워크 연결 등을 안전하게 정리하여 메모리 누수(Memory Leak)를 방지하는 핵심적인 역할을 합니다.

---

### 2. 지역 객체와 전역 객체의 비교

| 구분 | 지역 변수 (지역 객체, Local Object) | 전역 변수 (전역 객체, Global Object) |
| :--- | :--- | :--- |
| **선언 위치** | 함수 내부 또는 블록(`{ }`) 내부 | 모든 함수 외부 (주로 소스 파일 상단) |
| **접근 범위 (Scope)** | 선언된 블록 내부에서만 접근 가능 (Block Scope) | 프로그램 전체 (동일 파일 내 어디서든 접근 가능, Global Scope) |
| **생성 시점** | 실행 흐름이 **해당 객체의 선언문에 도달했을 때** | 프로그램 로딩 시점, 즉 **`main()` 함수가 시작되기 전** |
| **소멸 시점** | 선언된 블록(`}`)을 **벗어날 때** | 프로그램 종료 시점, 즉 **`main()` 함수가 종료된 후** |
| **초기화 여부** | 명시하지 않으면 **쓰레기값(Garbage Value)** 존재. (단, 클래스 객체는 기본 생성자 자동 호출) | 명시하지 않아도 **0으로 자동 초기화** (BSS 영역). (클래스 객체는 기본 생성자 자동 호출) |

---

### 3. 프로그램 실행 후 `main` 함수가 실행되기 전에 처리되는 일

운영체제(OS)가 프로그램을 메모리에 적재한 후, `main()` 함수가 호출되기 전 C++ 런타임(Runtime) 환경에서는 다음과 같은 초기화 작업이 일어납니다.

1. **메모리 공간 확보:** 프로그램 실행에 필요한 코드(Code), 데이터(Data), BSS, 스택(Stack), 힙(Heap) 영역이 메모리에 구성됩니다.
2. **전역 및 정적 변수 초기화:** 데이터(Data) 영역과 BSS 영역에 들어갈 전역 변수 및 정적(static) 변수들의 메모리를 할당하고 초기화합니다.
3. **전역 객체의 생성자 호출:** `main()` 함수 외부에 선언된 **모든 전역 객체들의 생성자가 선언된 순서대로 호출**됩니다.
4. **`main` 함수 진입:** 위 시스템 레벨의 초기화 작업이 모두 안전하게 끝난 후, 시스템이 `main()` 함수를 호출하여 본격적인 프로그램 로직이 시작됩니다.

---

### 4. `main` 함수 종료 후 프로그램이 종료하기 전에 처리되는 일

`main()` 함수가 `return` 문을 만나 종료되더라도 즉각적으로 프로세스가 소멸하는 것은 아닙니다. 다음과 같은 마무리 작업(정리 작업)이 진행됩니다.

1. **전역 및 정적 객체의 소멸자 호출:** 메모리(데이터 영역)에 남아있던 **전역 객체들의 소멸자가 생성된 순서의 역순(LIFO)으로 호출**됩니다.
2. **표준 입출력 스트림 버퍼 비우기 및 닫기:** `cin`, `cout` 등 프로그램이 사용하던 표준 입출력 버퍼(Buffer)를 비우고 스트림을 안전하게 닫습니다.
3. **메모리 해제 및 제어권 반환:** 프로세스 종료 상태 코드를 운영체제에 전달하고, 프로그램이 사용하던 전체 메모리 영역을 운영체제가 최종적으로 회수(Reclaim)합니다.

### 실행결과가 다른 이유
실습과제3의 경우 main() 가 실행되기 전에 전역 객체로 생성되어 프로그램 전체의 상태를 관리합니다.

지역 객체: main 함수(또는 특정 블록)라는 범위(Scope)에 종속됩니다. 함수가 끝나면 사라지는 일회용 소모품 같은 존재입니다.

전역 객체: 프로그램 전체의 상태를 담당합니다. 어떤 함수에서도 접근이 가능하며, 프로그램이 살아있는 내내 메모리를 점유하는 정적 수명(Static Storage Duration)을 가집니다.

객체가 메모리에 존재하는 시간대(Time Zone) 자체가 다르기 때문에 C++ 설계 원칙상 두 방식은 완전히 다른 코드로 간주됩니다.

따라서 되도록 지역 객체를 위주로 사용하되 공유가 필요한 객체의 경우 전역 객체를 상수로 관리하는 것이 일반적입니다
