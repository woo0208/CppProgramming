# 실습과제 1
### 1. 접근 지정자가 왜 필요한가?
접근 지정자(Access Specifier: public, private, protected)는 클래스의 멤버(변수 및 함수)에 대한 외부 코드의 접근 권한을 엄격하게 제어하기 위해 필요합니다. 
* **객체 상태의 무결성 보호:** 클래스 내부의 핵심 데이터(멤버 변수)를 무분별한 외부 직접 접근으로부터 차단하여, 객체의 상태가 설계자가 의도하지 않은 유효하지 않은 값으로 오염되는 것을 막습니다.
* **정보 은닉의 문법적 수단:** 객체의 내부 구현 디테일을 숨기고, 외부에는 안전하게 검증된 사용 방법(Public 인터페이스)만을 노출하게 함으로써 객체 지향의 핵심 원칙인 정보 은닉(Information Hiding)을 C++ 언어 차원에서 실현하는 문법적 도구입니다.

### 2. 생성자와 setter 멤버 함수의 차이점
두 기능 모두 객체의 멤버 변수에 값을 할당한다는 공통점이 있지만, **객체의 생명 주기 내 호출 시점**과 **근본적인 목적**에서 명확한 차이가 있습니다.
* **생성자(Constructor):**
    * **호출 시점:** 객체가 메모리에 할당되고 **생성(Instantiation)될 때 단 한 번, 시스템에 의해 자동으로 호출**됩니다.
    * **목적:** 객체가 메모리에 올라가 사용되기 전에 필요한 최초의 초기화(Initialization)를 담당합니다. 멤버 변수에 쓰레기값(Garbage Value)이 남지 않도록 초기 상태를 확립하거나 필요한 동적 메모리를 할당하는 것이 주 목적입니다.
* **setter 멤버 함수(Setter Member Function):**
    * **호출 시점:** 객체가 완전히 생성되어 메모리에 존재하는 동안, 프로그래머가 필요에 의해 **명시적으로 횟수 제한 없이 호출**할 수 있습니다.
    * **목적:** 객체의 생명 주기 도중 외부의 요청에 따라 내부 상태(멤버 변수의 값)를 변경(Modification)하는 것이 목적입니다. 주로 전달받은 인자(Argument)의 유효성을 검사한 후 안전하게 멤버 변수를 업데이트할 때 사용됩니다.

### 3. 캡슐화(Encapsulation)와 정보 은닉(Information Hiding)
이 두 개념은 상호보완적이며 함께 쓰이지만, 기술적으로 명확히 구분되는 의미를 가집니다.
* **캡슐화(Encapsulation):** 성격이 관련되어 있는 데이터(멤버 변수)와 그 데이터를 조작하고 처리하는 연산 로직(멤버 함수)을 분리하지 않고, 하나의 논리적인 단위인 클래스(Class)로 결합하여 묶는 행위를 의미합니다. 
* **정보 은닉(Information Hiding):** 캡슐화된 클래스를 설계할 때, 외부에서 직접 제어해서는 안 되는 데이터나 복잡한 내부 동작 알고리즘을 `private`으로 선언하여 숨기는 설계 원칙입니다. 대신 외부와의 소통은 철저히 `public`으로 공개된 멤버 함수(인터페이스)를 통해서만 이루어지도록 강제합니다. 이를 통해 외부 코드가 클래스 내부 구조에 의존하는 것을 막아 소프트웨어의 유지보수성과 확장성을 높입니다.
