# 실습과제 1

### 1. 구조체(struct)와 클래스(class)의 차이점
C++에서 구조체와 클래스는 멤버 변수와 멤버 함수를 가질 수 있고 상속도 가능하다는 점에서 기능적으로 거의 동일합니다. 유일한 차이점은 기본 접근 지정자(Default Access Specifier)에 있습니다.
* **구조체(struct)**: 명시적으로 접근 지정자를 선언하지 않으면 기본적으로 모든 멤버가 **`public`**으로 설정되어 외부에서 접근이 가능합니다.
* **클래스(class)**: 명시적으로 지정하지 않으면 기본적으로 모든 멤버가 **`private`**으로 설정되어 외부 접근이 차단됩니다. 

C++ 객체 지향 프로그래밍에서는 데이터의 무결성을 보호하는 **캡슐화(Encapsulation)** 및 정보 은닉(Information Hiding)이 중요하므로, 기본적으로 내부 데이터를 보호하는 `class`의 사용을 원칙으로 합니다.

### 2. 클래스와 객체의 차이점
추상적인 비유(예: 붕어빵 틀과 붕어빵)를 배제하고 메모리와 데이터 관점에서 설명하겠습니다.
* **클래스(Class)**: 개발자가 설계하는 사용자 정의 데이터 타입(User-Defined Data Type)입니다. 메모리에 직접 할당되지 않으며, 해당 타입이 가져야 할 속성(멤버 변수)과 행위(멤버 함수)를 선언하는 컴파일 타임의 명세서(Specification)입니다.
* **객체(Object / 인스턴스 Instance)**: 클래스라는 데이터 타입을 기반으로 런타임에 **실제 메모리가 할당된 실체**입니다. 프로그램 실행 중 메모리(스택이나 힙 영역)에 생성되어 고유한 데이터를 저장하고, 선언된 멤버 함수를 통해 상태를 변경하거나 연산을 수행합니다.


### 3. sizeof(클래스명)의 결과값과 그 의미
`sizeof(클래스명)`은 해당 클래스 타입으로 객체를 생성했을 때, 그 객체가 메모리상에서 차지하는 **바이트(byte) 크기**를 반환합니다. 결과값은 다음 규칙에 의해 결정됩니다.
1. **비정적(non-static) 멤버 변수의 합**: 객체의 크기는 오직 클래스 내부에 선언된 일반 멤버 변수들의 크기 합으로 구성됩니다. 멤버 함수 코드나 정적(static) 멤버 변수는 객체 외부에 별도로 존재하므로 객체의 크기에 포함되지 않습니다.
2. **메모리 패딩 및 정렬(Memory Alignment / Padding)**: CPU가 메모리에 빠르게 접근하도록 돕기 위해, 컴파일러는 멤버 변수들 사이에 의미 없는 빈 공간(Padding)을 추가하여 일정한 바이트 배수(예: 4바이트, 8바이트)로 메모리 주소를 정렬합니다. 따라서 단순히 멤버 변수 크기를 더한 것보다 결과값이 더 크게 나올 수 있습니다.
3. **가상 함수 포인터(vptr)**: 클래스 내에 `virtual` 함수가 선언되어 있다면, 가상 함수 테이블(vtable)을 가리키는 숨겨진 포인터(32비트 환경 4바이트, 64비트 환경 8바이트)가 객체 크기에 추가됩니다.
4. **빈 클래스**: 멤버가 전혀 없는 빈 클래스의 크기는 0이 아니라 **1바이트**입니다. C++은 서로 다른 두 객체가 동일한 메모리 주소를 가지는 것을 방지하기 위해 최소 1바이트를 할당합니다.
