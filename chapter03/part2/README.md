# 실습과제 1

### 1. C++에서 생성자를 통해 멤버 변수를 초기화하는 이유

객체(Object)가 인스턴스화(Instantiation)될 때, 즉 메모리에 공간이 할당될 때 초기화되지 않은 멤버 변수에는 의미 없는 쓰레기값(Garbage value)이 들어가게 됩니다.

* **안전성과 무결성 보장:** 생성자(Constructor)는 객체가 생성되는 시점에 컴파일러에 의해 자동으로 호출되는 특별한 멤버 함수입니다. 생성자를 통해 멤버 변수에 적절한 유효 값을 할당함으로써, 객체가 생성됨과 동시에 정상적으로 작동할 수 있는 안전한 상태(Valid state)를 보장하기 위해 사용합니다.
* **캡슐화(Encapsulation) 원칙 준수:** 외부에서 직접 멤버 변수에 접근하여 값을 넣는 것은 객체 지향의 정보 은닉에 위배됩니다. 생성자를 통하면 외부에서는 초기화에 필요한 값만 전달하고, 내부적인 설정은 클래스 스스로 처리하게 할 수 있습니다.

### 2. 생성자 몸체(Body) 대입보다 멤버 초기화 리스트(Initialization List)를 사용하는 이유

* **성능 상의 이점 (초기화 vs 생성 후 대입):**
    * **몸체에서 초기화:** 객체의 멤버가 먼저 '디폴트 생성자'에 의해 생성된 후, 몸체 안에서 새로운 값으로 대입(Assignment)됩니다. (즉, 기본 객체 생성 1번 + 대입 연산 1번 = 총 2번의 작업)
    * **초기화 리스트 사용:** 객체가 메모리에 생성됨과 동시에 해당 값으로 바로 초기화(Direct Initialization)됩니다. 불필요한 디폴트 생성자 호출과 대입 연산을 막아 성능이 크게 향상됩니다.
* **문법적 필수성 (반드시 초기화 리스트를 써야만 하는 경우):**
    * **상수(const) 멤버 변수**와 **참조자(Reference) 멤버 변수**는 선언과 동시에 초기화되어야 하므로, 생성자 몸체 내부에서의 대입이 문법적으로 불가능하며 반드시 초기화 리스트를 사용해야 합니다.
    * 디폴트 생성자가 없는 기본 클래스(Base Class)를 상속받았을 때나, 멤버로 포함된 다른 클래스 객체가 디폴트 생성자를 제공하지 않을 때도 초기화 리스트를 통해서만 매개변수를 전달할 수 있습니다.

### 3. 위임 생성자(Delegating Constructor)와 타겟 생성자(Target Constructor)로 나누어 작성하는 이유

* **코드 중복 제거 (Eliminating Code Duplication):** 하나의 클래스에 다양한 매개변수를 받는 여러 개의 오버로딩된 생성자가 있을 때, 각 생성자마다 공통된 초기화 코드를 중복해서 작성하면 코드가 길어지고 비효율적입니다.
* **유지보수성 향상:** C++11 표준부터 도입된 이 기능을 사용하면, 객체 초기화의 핵심 로직을 하나의 **타겟 생성자**에 몰아두고, 다른 생성자들은 이 타겟 생성자에게 초기화 작업을 떠넘기는 **위임 생성자** 역할을 하게 됩니다. 이를 통해 초기화 로직이 변경되더라도 타겟 생성자 한 곳만 수정하면 되므로 오류 발생 확률이 줄어들고 유지보수가 매우 쉬워집니다.

### 4. 함수 몸체(Body)에서 생성자를 직접 호출하면 일어나는 일

다른 프로그래밍 언어의 경험 때문에, 이미 생성된 현재 객체의 멤버 변수를 다시 초기화하기 위해 멤버 함수나 생성자 내부에서 `ClassName();`과 같이 생성자를 직접 호출하는 실수를 종종 합니다.

* **이름 없는 임시 객체(Unnamed Temporary Object) 생성:** C++에서 생성자를 명시적으로 호출하면, 현재 객체(this)가 재초기화되는 것이 **절대 아닙니다**. 대신, 컴파일러는 메모리 어딘가에 자신만의 공간을 가지는 이름 없는 새로운 임시 객체를 하나 더 생성합니다.
* **즉각적인 소멸:** 이 임시 객체는 해당 실행 문장(Statement)이 끝나는 즉시(세미콜론 `;`을 만나는 순간) **소멸자(Destructor)**가 호출되며 메모리에서 사라집니다. 결과적으로 원래 의도했던 현재 객체의 멤버 변수 값은 전혀 변하지 않으며, 불필요한 객체의 생성과 소멸로 인해 컴퓨팅 리소스만 낭비하게 됩니다.
